Chapter 9 객체와 클래스


1. 객체를 위한 클래스 정의

* 객체 : 명확하게 구별되는 실제 세계에서의 개체(요소)를 나타낸다. 자신만의 유일한 특성과 상태, 행동을 가진다.

- 객체의 상태/속성 : 현재 값을 가지고 있는 데이터 필드로 표현됨

- 객체의 행동/동작 : 함수에 의해 정의됨, 객체에 대한 함수를 호출하는 것은 객체에 어떤 동작을 수행하도록 요구하는 것

같은 유형의 객체는 공통 클래스를 사용하여 정의된다. 클래스(class)는 객체의 데이터 필드와 함수가 무엇이 될지를 결정하는 템플릿(template) 또는 설계도, 규약이라고 할 수 있으며 객체는 클래스의 인스턴스(instance)이다.

* 실체화(instantiation) : 인스턴스를 생성하는 것 (종종 객체와 인스턴스라는 용어가 같은 의미로 사용되기도 한다)

* C++ 클래스는 데이터 필드를 정의하기 위해 변수를 사용하며, 행동을 정의하기 위해 함수를 사용한다. 클래스는 생성자(constructor)(특별한 유형의 함수)를 제공, 생성자는 새로운 객체가 생성될 때 호출된다.

* 클래스와 객체는 UML(Unified Modeling Language) 표기법을 사용해 표현 가능 -UML 클래스 다이어그램/클래스 다이어그램

데이터 필드는 dataFieldName: dataFieldType으로 표시, 생성자는 ClassName(parameterName: parameterType)으로 표시, 함수는 functionName(parameterName: parameterType): returnType으로 표시


2. 생성자

* 생성자(constructor) : 함수의 특별한 유형, 세 가지 특성

- 생성자는 클래스 자신과 같은 이름을 가져야 한다.

- 생성자는 반환 유형이 없다. (void도 사용할 수 없다.)

- 생성자는 객체가 생성될 때 호출된다. 생성자는 객체 초기화의 역할을 수행한다.

클래스는 일반적으로 인수가 없는 생성자(no-argument constructor)를 제공한다.

클래스는 생성자 없이 정의될 수도 있다. 내용(body)이 비어 있는 인수 없는 생성자가 암시적으로 클래스 내부에서 정의되는데, 이를 기본 생성자(default constructor)라 하며, 클래스 안에 어떠한 생성자도 명시적으로 정의되지 않을 경우에만 자동으로 제공된다.


3. 객체 구성과 사용

인수 없는 생성자를 사용하여 객체를 생성하기 위한 구문

ClassName objectName;

인수 있는 생성자를 사용하여 객체를 생성하기 위한 구문

ClassName objectName(인수);

객체의 멤버(member) : 데이터 필드와 함수를 의미

새롭게 생성된 객체는 메모리에 할당된다. 객체 생성된 후에 객체 멤버 접근 연산자(object member access operator)라고 하는 점 연산자(.)를 이용해 데이터에 접근하고 함수를 호출할 수 있다.

* 클래스와 객체에 대한 주의사항

- 변수 선언하는데 원시(primitive) 데이터 유형 사용 가능

- C++에서 하나의 객체로부터 다른 객체로 내용 복사하기 위해 대입 연산자(=) 사용 가능

- 객체 이름은 배열 이름과 같다. 객체의 내용은 변할 수 있지만 객체 이름은 변경할 수 없다.

- 객체는 데이터를 포함하며 함수를 호출할 수 있다.


4. 클래스 정의와 구현 분리

클래스 정의 : 클래스의 규약 사항을 기술하는 것, 모든 데이터 필드와 생성자 원형(prototype), 함수 원형을 목록으로 만드는 것

클래스 구현 : 그 규약을 실현하는 것, 생성자와 함수를 만드는 것

클래스 정의 파일의 확장자는 .h(h는 헤더를 의미), 클래스 구현 파일의 확장자는 .cpp이다.

:: 기호는 이항 범위 지정 연산자(binary scope resolution operator)로, 클래스에서 클래스 멤버의 범위를 나타내 준다.


5. 다중 포함 방지

헤더 파일이 여러 번 포함되는 것을 방지하기 위해서 포함 감시를 사용한다.

#define과 함께 사용되는 #ifndef 지시자(directive)는 헤더 파일이 여러 번 포함(include)되는 것을 방지하는 데 사용되며, 이를 포함 감시(inclusion guard)라고 한다.

Class, object, constructor


객체 지향 : 객체들을 조립해서 신속하게 소프트웨어 만들 수 있다

Object

객체 : 상태(속성, 값)와 동작(기능, 역할)으로 표현되는 어떤 것

Class

클래스 : 객체를 만드는 설계도(클래스는 1개, 객체는 여러 개)

클래스는 변수(데이터 정의), 함수(행동 정의)를 사용한다.

클래스는 객체의 '데이터 유형'이다.

Constructor

생성자(constructor)(특별한 유형의 함수, 클래스와 동일한 이름을 가진 함수)는 클래스에서 제공한다. 생성자는 클래스로부터 객체가 생성될 때 호출된다.

UML(Unified Modeling Language) : 클래스와 객체 정의를 UML로 표현

생성자의 특성

* 생성자는 클래스 자신과 같은 이름을 가져야 한다.

* 생성자는 반환 유형이 없다. (void도 사용할 수 x)

* 생성자는 객체가 생성될 때 호출된다.

* 생성자는 객체 초기화의 역할을 수행한다.

- 클래스의 변수는 선언될 때 초기화할 수 없다. (코드에서 객체 선언할 때, 객체가 생성되면서 생성자가 호출되고 변수를 저장할 메모리 공간도 할당된다.)

* 생성자도 오버로딩이 가능하다. (이름은 같지만 매개변수 형태가 다른 생성자 여러 개 정의 가능)

생성자 없는 클래스

*생성자는 생략 가능하다.

*기본 생성자(default constructor) : 생성자 없는 클래스의 경우, 내용(body)이 비어 있는 인수 없는 생성자가 암시적으로 클래스 내부에서 정의

클래스 안에 어떠한 생성자도 명시적으로 정의되지 않을 경우에만 자동적으로 제공된다.
