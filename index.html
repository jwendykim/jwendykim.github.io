교재 C++로 시작하는 객체지향 프로그래밍 9장 요약

Chapter 9 객체와 클래스

1. 객체를 위한 클래스 정의

* 객체 : 명확하게 구별되는 실제 세계에서의 개체(요소)를 나타낸다. 자신만의 유일한 특성과 상태, 행동을 가진다.

- 객체의 상태/속성 : 현재 값을 가지고 있는 데이터 필드로 표현됨

	
- 객체의 행동/동작 : 함수에 의해 정의됨, 객체에 대한 함수를 호출하는 것은 객체에 어떤 동작을 수행하도록 요구하는 것

	
같은 유형의 객체는 공통 클래스를 사용하여 정의된다. 클래스(class)는 객체의 데이터 필드와 함수가 무엇이 될지를 결정하는 템플릿(template) 또는 설계도, 규약이라고 할 수 있으며 객체는 클래스의 인스턴스(instance)이다.

	
*실체화(instantiation) : 인스턴스를 생성하는 것 (종종 객체와 인스턴스라는 용어가 같은 의미로 사용되기도 한다)

	
*C++ 클래스는 데이터 필드를 정의하기 위해 변수를 사용하며, 행동을 정의하기 위해 함수를 사용한다. 클래스는 생성자(constructor)(특별한 유형의 함수)를 제공, 생성자는 새로운 객체가 생성될 때 호출된다.

	
*클래스와 객체는 UML(Unified Modeling Language) 표기법을 사용해 표현 가능
		-UML 클래스 다이어그램/클래스 다이어그램

	
 데이터 필드는 dataFieldName: dataFieldType으로 표시, 생성자는 ClassName(parameterName: parameterType)으로 표시, 함수는 functionName(parameterName: parameterType): returnType으로 표시

	
2. 생성자

	
*생성자(constructor) : 함수의 특별한 유형, 세 가지 특성

	
- 생성자는 클래스 자신과 같은 이름을 가져야 한다.

	
- 생성자는 반환 유형이 없다. (void도 사용할 수 없다.)

	
- 생성자는 객체가 생성될 때 호출된다. 생성자는 객체 초기화의 역할을 수행한다.

	
클래스는 일반적으로 인수가 없는 생성자(no-argument constructor)를 제공한다.

	
클래스는 생성자 없이 정의될 수도 있다. 내용(body)이 비어 있는 인수 없는 생성자가 암시적으로 클래스 내부에서 정의되는데, 이를 기본 생성자(default constructor)라 하며, 클래스 안에 어떠한 생성자도 명시적으로 정의되지 않을 경우에만 자동으로 제공된다.

	
3. 객체 구성과 사용

	
인수 없는 생성자를 사용하여 객체를 생성하기 위한 구문

	
ClassName objectName;

	
인수 있는 생성자를 사용하여 객체를 생성하기 위한 구문

	
ClassName objectName(인수);
